## 1.- LISTAS NO ORDENADAS (VIÑETAS)
Para usar este formato de texto es necesario usar ( - )( * )( + )
para textos donde el orden no sea prioridad

+ comida
+ estudio

### ANIDAMIENTO

ESTE ANIDAMIENTO EN UNORDERED LIST FUNCIONA GRACIAS A LA TECLA DE TABULACION 

- HOLA
	- ANIDADO
	- ANIDADO
		- MAS ANIDADO AUN
	- SIP
- NOP
## 2.- LISTAS ORDENADAS

Para este formato de texto se usa la sintaxis de un numero seguido de un punto 1. y un espacio obviamente

1. javascrip
2. framework
3. backend

### ANIDAMIENTO

EN ESTE CASO FUNCIONA CON LA TABULACION AL IGUAL QUE EN LA LISTAS NO ORDENADAS

1. HOLA
2. COMO
	1. ESTAS
	2. TODO
	3. BIEN?
3. SIP 
4. TODO 
5. BIEN
## 3.- LISTA DE TAREAS CHECKLISTS 

Esta es una caracteristica heredada de github que funcina como una lista normal desordenada con una casilla de verificacion
La sintaxis se realiza con un guion seguido de un espacio y corchetes vacios con un espacio en caso de estar *incompletas* o corchetes con una x en caso de estar *completas*

```
- [ ] tarea cheklist incompleta
- [x] tarea completa
```

- [ ] realizar fase dos de markdown
- [x] empezar fase dos

## 4.- CITAS EN BLOQUE

CUANDO QUIERAS CITAR, REFERENCIAR A UN AUTOR O HACER UNA ADVERTENCIA SE APLICA ESTE FORMATO 
LA SINTAXIS ES UN MAYOR QUE SEGUIDO DE UN ESPACIO Y EL TEXTO QUE DESEES

```
> cita textual en bloque
```

> comida 25
> este es un texto citado en bloque
